# The Vine Solutions AI — Client Onboarding Checklist

**Client:** ____________________   **Type:** ☐ E-commerce  ☐ Service  ☐ Other
**Start date:** ____________   **Owner (TVS):** ____________   **Target go-live:** ____________

> Focus order: **1) CRM (core)** → 2) Website build (if scoped) → 3) Handover & optimization.
> Check every box before advancing to the next phase.

---

## Phase 0 — Sales-to-Delivery Handover (Day 0)
- [ ] Signed proposal / SOW on file (scope, price, timeline)
- [ ] Deposit / first invoice paid
- [ ] Scope confirmed: CRM only, or CRM **+ website build**
- [ ] Primary client contact + decision-maker identified (name, email, phone)
- [ ] Kickoff call booked (calendar invite sent)
- [ ] Shared drive / project folder created for this client
- [ ] Onboarding form / intake questionnaire sent to client

---

## Phase 1 — Discovery & Access
- [ ] Business overview captured (what they sell, avg order value, monthly volume)
- [ ] Sales process mapped (how a lead becomes a customer, current tools)
- [ ] Team roster + roles (who needs CRM logins and at what permission level)
- [ ] Current data source identified (spreadsheet, old CRM, Shopify export, etc.)
- [ ] Access / credentials collected:
  - [ ] Email/domain provider
  - [ ] E-commerce platform (Shopify / WooCommerce / etc.)
  - [ ] Existing CRM export (CSV) if migrating
  - [ ] Website hosting/domain registrar (if doing website)
  - [ ] Social / ad accounts (if lead sources feed CRM)
- [ ] Goals & KPIs agreed (e.g. faster follow-up, pipeline visibility, cart-abandon recovery)

---

## Phase 2 — CRM Setup (CORE — main deliverable)

### 2.1 Foundation
- [ ] CRM workspace / instance provisioned for client
- [ ] Company branding applied (logo, name, colors)
- [ ] User accounts created + roles/permissions assigned
- [ ] Two-factor / login security enabled

### 2.2 Data Model
- [ ] **Contacts** structure defined (name, email, phone, source, tags)
- [ ] **Companies / Accounts** structure defined
- [ ] **Opportunities / Deals** fields defined (value, close date, owner)
- [ ] Custom fields added for the client's needs
- [ ] **E-commerce fields** added *(ecommerce clients)*:
  - [ ] Sales channel (Direct / Shopify / Amazon / Marketplace / Social)
  - [ ] Order value / LTV
  - [ ] Product / SKU interest
  - [ ] Cart status (abandoned / purchased / repeat)

### 2.3 Pipeline
- [ ] Pipeline stages defined and named for the client's process
- [ ] E-commerce pipeline template applied *(e.g. Lead → Engaged → Cart → Checkout → Won → Repeat)*
- [ ] Stage automation rules confirmed (what moves a deal forward)
- [ ] "Lost" reasons list configured

### 2.4 Data Migration
- [ ] Existing contacts/deals imported and de-duplicated
- [ ] Data validated (spot-check 10–20 records)
- [ ] Tags / segments applied to imported records
- [ ] Owners assigned to migrated records

### 2.5 Automation & Integrations
- [ ] Email inbox connected (send/receive from CRM)
- [ ] E-commerce platform connected *(ecommerce clients — orders/customers sync in)*
- [ ] Lead-capture sources wired (website forms, ads, chat)
- [ ] Automated follow-up sequences set up (welcome, abandoned cart, post-purchase)
- [ ] Task/reminder rules for the sales team
- [ ] Notifications tested (new lead, deal won, etc.)

### 2.6 Reporting
- [ ] Dashboard configured (pipeline value, deals by stage)
- [ ] **Revenue-by-sales-channel chart** enabled *(ecommerce clients)*
- [ ] Conversion / win-rate reporting set up
- [ ] Activity reporting for team accountability

### 2.7 CRM Testing & Sign-off
- [ ] End-to-end test: create lead → move through pipeline → close
- [ ] Search, filter, and tags verified
- [ ] Permissions tested per role
- [ ] Automations fire correctly (test email/order)
- [ ] Client walkthrough completed
- [ ] **Client sign-off on CRM** ✅

---

## Phase 3 — Website Build (if scoped)

### 3.1 Planning
- [ ] Sitemap / page list agreed (Home, Shop, Product, About, Contact, etc.)
- [ ] Brand assets collected (logo, colors, fonts, images, copy)
- [ ] Reference sites / design direction gathered
- [ ] Domain + hosting confirmed (registrar access, DNS plan)

### 3.2 Build
- [ ] Design mockup / theme approved by client
- [ ] Pages built and responsive (mobile + desktop)
- [ ] E-commerce storefront configured *(products, pricing, checkout, payment gateway)*
- [ ] **Lead forms wired to CRM** (this is the CRM ↔ website link — critical)
- [ ] Tracking installed (analytics, pixel, conversion events)
- [ ] SEO basics (titles, meta, sitemap, alt text)
- [ ] Legal pages (privacy, terms, returns/shipping for ecommerce)

### 3.3 Launch
- [ ] Cross-browser + mobile QA passed
- [ ] Page speed / performance checked
- [ ] Forms + checkout tested end-to-end (order lands in CRM)
- [ ] DNS pointed / domain live over HTTPS (SSL)
- [ ] Client sign-off on website ✅

---

## Phase 4 — Handover & Go-Live
- [ ] Training session delivered (CRM usage for the team)
- [ ] Quick-reference guide / Loom videos shared
- [ ] Admin credentials + password manager entry handed over
- [ ] Support channel + response times explained
- [ ] Maintenance / retainer plan confirmed (if applicable)
- [ ] 30-day check-in scheduled
- [ ] Final invoice sent
- [ ] Testimonial / referral requested

---

## Post-Launch (Ongoing)
- [ ] Week 1: monitor automations + data flow
- [ ] Week 2: review adoption, answer questions
- [ ] Day 30: KPI review vs. Phase 1 goals
- [ ] Identify upsell / optimization opportunities
