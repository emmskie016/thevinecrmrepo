Project Overview

Goal

Turn the client-side Vine CRM prototype into a multi-tenant SaaS: real auth, Postgres persistence with RLS as the security boundary, CI/CD, plus a Conversations inbox (email/SMS) and email campaigns.

Locked decisions

  • Mono-repo in new GitHub org vine-solutions
  • Multi-tenant: org_id everywhere, roles owner/admin/member/viewer
  • Keep Vite SPA + supabase-js (no separate API)
  • Email via Gmail OAuth (2-way); SMS provider-agnostic for now

Tracks

  • frontend UI, live-data hooks, inbox & campaign views
  • backend schema, RLS, edge functions, Gmail/SMS adapters
  • fullstack repo/CI/deploy, auth, cross-cutting wiring

Risks / constraints

  • Gmail send caps (~500–2,000/day) limit campaign volume
  • Gmail Pub/Sub inbound needs GCP setup (fallback: cron sync)
  • OAuth tokens must stay server-side (Vault + edge functions)
  • RLS bootstrap chicken-and-egg → create_org_with_owner RPC
